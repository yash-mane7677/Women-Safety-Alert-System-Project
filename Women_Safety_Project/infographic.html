Technology Breakdown

This system is a full-stack application. The frontend is built for immediate user interaction, while the backend handles secure and reliable alert dispatch.

Multi-Channel Alerts

The system sends alerts through two primary channels to ensure redundancy and reliable delivery to emergency contacts.

System Process Flow

Click on each step below to learn more about how the system works, from a user action to a sent alert.

Step 1: User Action & Confirmation

The process begins when the user presses the SOS button on the web interface. A confirmation modal is triggered to prevent accidental alerts, ensuring the user's intent is confirmed before the system proceeds.

Step 2: Geolocation & Data Capture

Once confirmed, the frontend's JavaScript logic activates. It utilizes the browser's built-in Geolocation API to request and capture the device's precise latitude and longitude. This location data is critical for the next steps.

Step 3: API Call to Backend

With the location coordinates secured, the frontend sends a secure `POST` request to the backend server's API endpoint. The request body contains the location data in a structured JSON format, ready for the server to process.

Step 4: Multi-Channel Alert Dispatch

The Python Flask server receives the API call. It instantly initiates the multi-channel alert protocol, using different libraries (e.g., `smtplib`, `requests`) to dispatch a predefined alert message to all configured emergency contacts via email and Telegram.
